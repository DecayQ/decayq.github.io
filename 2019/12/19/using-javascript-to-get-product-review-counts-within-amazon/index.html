上一次写给女友写的书签脚本已经过去了快半年了，可以参考文章。其实中间断断续续地又写了三四个，只不过差别不大就没有记录。今天中午的时候给了一个新的需求，刚一看蛮有挑战性的就做了。顺便在这里记录一下思路。

题目是这样，作为运营的女友需要从商品网页上获取商品的评价数量，但是每个商品页里面又有不同的 pack（姑且先这么叫），那不同的 pack 下的 review 数量也不相同。她现在的做法是点开某个具体的 pack 后拉动到页面下方点击跳转到评论页后选中当前 format 后才能看到当前 pack 下的评论数。那怎样才能快速获取呢？

第一个想法是：既然每个 pack 都需要跳转到评论页才能看，那么有多少个 pack 就打开多少个对应的评论页好了，用空间换时间。而且选中下方的过滤条件后 url 也是有变化的，所以我们只要拿到包含了过滤条件的 url，那么一打开就能直接显示当前 pack 的评论数据。

分析评论页的 url 后，我分成这么几个部分：

  1. HOST:
    https://www.amazon.com/
  2. 商品名称：
    Product-Name/
  3. 商品评价：
    product-reviews/
  4. ASIN:
    B081H1PGJQ/
  5. 过滤和其他参数：
    ref=cm_cr_arp_d_viewopt_fmt?formatType=current_format&pageNumber=1

其中只有商品名称和 ASIN 是相关字段，其他的都可以硬编码。那么接下来就是如何获取这两个参数的过程。
